# Тестовое задание

**Интеллектуальный менеджер задач с интеграцией LLM**

## Обязательное содержание README:

1. Краткое описание проекта и реализованных функций.<br>
   Можно создавать, читать, обновлять и удалять задачи. Каждая задача имеет название, описание, приоритет (низкий/средний/высокий), статус (ожидает/в работе/готово), срок выполнения и время создания. 
   Можно фильтровать задачи по статусу, приоритету и сроку. Также искать задачи по названию или описанию через поле текстового поиска. 
2. Пошаговые инструкции настройки среды (зависимости, установка, переменные окружения).<br>
    `.env` файл находится в репозитории, так как не содержит ничего важного<br>
    Тут 2 варианта, запустить dev версию и стартовать базу с докера или просто запустить докер:<br>
   1. `npm run dev` `docker compose up postgres`<br>
   2. `docker compose up --build`<br>
   И в первом и во втором случае нужно запустить миграции<br>
   `npm run migrate:dev` или `npm run migrate:deploy`<br>
3. Инструкции запуска приложения (фронтенд и бэкенд).<br>
    Запуск описал выше, нужно стартовать next.js приложение, базу в докере и миграции запустить
4. Описание принятых архитектурных решений.<br>
    Приложение написано на next.js и prisma, так как это очень удобный стэк для стартапов и легко всё типизировать. Проект типизирован 100%
5. Известные проблемы, ограничения или компромиссы.<br>
    Ограничений нет, внизу написал что еще планирую добавить
6. Список функций, которые кандидат добавил бы при наличии дополнительного времени.<br>
   Не сделал часть с искусственным интеллектом, но в дальнейшем займусь. Надеюсь что успею до того как тестовое проверят.
